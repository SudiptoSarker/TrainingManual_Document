LMS - License Management System

Introduction :


Overview

License Management System, that manages different services for Mopita site. This system simplifies the process to integrate a new service site into system. This system helps planners to design a webpage and insert the data into system. This system allows planners to set dynamic data of web page design so that engineers can easily develop web pages in code base. This system also operates service site. Service site allows user for a secure subscription process for a planner. This system also reduces engineers work. Engineers use AI tool to convert the design into code and utilizes planners dynamic data to make the web pages dynamic

In a summary, this system simplifies Mopita to integrate a service and planner can easily set dynamic data. Also reduce the development task by utilizing AI tool.

Installation :


Production installation

There are several processes to deploy project on Azure services. In this project deployment is done using tools like Microsoft SQL Server Management Studio and IDE (VS code).

  • Web App and Database Creation
    • Create Web App:
      • At first login to Azure Portal: Azure Portal
        1. Select a subscription
        2. Select a resource group
        3. Input instance name
        4. Select publish type
        5. Select runtime stack
        6. Select operating system
        7. Select region
        8. Select Linux plan
        9. Select Pricing plan
        10. Click review + create button

    • Create an Azure SQL Server:
      • Step-1:
        1. At first login to Azure Portal: Azure Portal
      • Step-2:
        1. In the Azure Portal, click on “Create a resource” and search for “SQL Server.”

        2. Select “SQL Server (logical server)” from the search results and click on “Create.”

        3. Choose the desired Subscription, Resource group (create a new one or select an existing one), Server name (should be unique) and Location.

        4. Set the Azure AD admin.

        5. Select the Authentication method and add the server admin login and password.
        6. Configure the additional settings, such as version, performance, and networking.
        7. Click on “Review + Create” and then “Create” to provision the Azure SQL Server. Wait for some time, as the azure will take some time for the deployment of the SQL Server.
      • Step-3:
        • Configure firewall settings for Azure SQL Server
          1. In the Azure Portal, navigate to the resource group containing your SQL Server.
          2. Click on the SQL Server resource to open its overview page.
          3. In the left menu, under “Security,” click on “Networking”.
          4. Click on “Add your client IPv4 address” to add your current IP address to the firewall rules.
          5. If you require access from specific IP ranges or Azure services, you can configure additional firewall rules accordingly.
          6. Click on “Save” to apply the changes.
      • Step-4:
        • Create an azure database
          1. In the Azure Portal, navigate to your SQL Server resource.
          2. Click on “Create database” to initiate the database creation process.
          3. Provide a name for the database, select the workload environment, Compute + storage. For the testing, I have selected the Standard-series (Gen5).
          4. Desired source (blank database or sample), and configure additional settings such as collation and security.
          5. Click on “Create” to create the Azure Database. Wait for some time until azure finishes the deployment of the database.
      • Step-5:
        • Connect to Azure SQL Server using SSMS
          To manage your Azure SQL Server and databases, you can use SQL Server Management Studio. Here’s how to connect to your Azure SQL Server using SSMS:
          1. Download and install SQL Server Management Studio from the official Microsoft website (Download SQL Server Management Studio).
          2. Open SSMS and click on “Connect” to open the “Connect to Server” window.
          3. Fill in the necessary details.
          4. Click on “Connect” to establish a connection with your Azure SQL Server.
      • Step-6:
        • Upload DB to server
          1. Download script from this link below (Database Script)
          2. Run scripts by clicking on the execute button.

Deploy Web App to Azure Server

  1. Request through support mail to get access for private git repository.
    • Send email
  2. After getting the access, download code and open in Visual Studio Code.
  3. Change some configuration in the projects.
    1. For License Management System, change in the .env file located in the project root folder.
      • NEXT_PUBLIC_DB_USER (set DB user)
      • NEXT_PUBLIC_DB_PASSWORD (set DB password)
      • NEXT_PUBLIC_DB_SERVER (set DB server)
      • NEXT_PUBLIC_DB_DATABASE (set DB name)
    2. For Bangla LMS, change in the .env file located in the project root folder.
      • NEXT_PUBLIC_DB_USER (set DB user)
      • NEXT_PUBLIC_DB_PASSWORD (set DB password)
      • NEXT_PUBLIC_DB_SERVER (set DB server)
      • NEXT_PUBLIC_DB_DATABASE (set DB name)

  4. Install Azure tools extension in VS Code, it will install all the dependent extensions.

  5. Click on the Azure Icon from the left panel and sign in if not signed in yet.

  6. After singing in you can see the resource list

  7. Expand desired resource, then expand App Service.

  8. Select desired app service, then right click on the mouse and select Deploy to Web App option.

  9. Expand your desired service and then expand Deployments option to check deployment status.
